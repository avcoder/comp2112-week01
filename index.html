nonblocking.js


const fs = require('fs');

// load food list asynchronously
fs.readFile('food.txt', 'utf8', function (err, food) {

    console.log(food);
  }
});

    console.log(err);
  } else {
    console.log(drinks);
  }
});

You asked...


  • Why nonblocking.js do not do what we anticipate? Why we have to run again and again the following code?
  • Why is Javascript an asynchronous language?
  • What is Callback function?

Answers...


  • Because we are using asynchronous operations. We only ran the code again and again to prove to ourselves that the output is not predictable
  • Because JS uses a call stack (that lives in V8), WebAPIs (that live in the browser), callback queue, and an event loop
  • Callback functions are just a normal JavaScript function that is called when an asynchronous operation is finished.
